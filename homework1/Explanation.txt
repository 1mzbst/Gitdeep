1. Инициализирую git: git init;
2. Проверяю статус гита: git status;
3. Создаю проект, добовляю его в гит для первого коммита: git add .\Название папки с проектом;
4. Делаю первый commit: git commit -m "Название коммита";
5. Меняю имя основной ветки master на main: git branch -M master main; 
6. Проверяю состояние веток: git branch;
7. Создаю новый репозитория на GITHUB: серия упорядочных нажатий на клавиши мышки преводят к созданию
    нового репозитория с таким же названием как и локальный. Создаю Readme.txt, делаю в нем правки, комичу его.
8. Копирую ссылку нахождения удалённого репозитория.
9. Добовляю связь удалённого репозитория с локальным: git remote add origin ссылка;
10. Проверяю статус origin: git remote -v;
    Далее, в моём случае существуют, два пути реализации загрузки локальных файлов в удалённый репозиторий:
1. Используя команду  git push -f -u ссылка названия основной ветки. Эта способ будет игнорировать находящиеся в
    удалённом репозитории файлы и перезапишет, находящиеся там данные, файлами из локального репозитория.
2. Второй способ это затянуть из удалённого репозитория файлы на локальный репозиторий при помощи команды
     git pull ссылка название ветки. Решить проблему с merge если потребуется, сделать коммит изменений и затем всё
     загрузить на удалённый репозиторий при помощи обычной команды git push.
Я выбрал первый вариант, так как на удалённом репозитории из файлов только README, не играющий роли.
